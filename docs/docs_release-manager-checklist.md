# Release Manager Checklist

Purpose:
- Standardize release readiness and reduce rollout risk.

Pre-release
- [ ] Release owner & scope confirmed
- [ ] Release notes drafted and reviewed
- [ ] Rollback plan documented and tested if feasible
- [ ] Monitoring & alerting configured for key signals
- [ ] Support/On-call notified and runbook available

Deployment
- [ ] Deployment window scheduled (if needed)
- [ ] Smoke tests and post-deploy verifications planned
- [ ] Stakeholder announcement prepared

Post-release
- [ ] Post-release verification completed
- [ ] Metrics dashboard checked vs. success criteria
- [ ] Incident retro or action items created if issues occur
